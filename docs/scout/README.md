# Scout Entries

Create one file per substantive scout mission using a stable three-digit id and slug:

```text
docs/scout/scout-001-projector-camera-hardware-baseline.md
```

Each scout should answer:

- what source or question was investigated,
- what evidence was found,
- whether the project should adopt, adapt, defer, reject, or spike,
- which spec/story/ADR surface owns the follow-up.

Check `docs/scout.md` before assigning a number.
